参考

WebSpec2Doc の設計・実装が依拠している国際標準と先行研究、および参考にした事例の一覧です。

実装に反映 は本製品の生成物・判定ロジックに直接反映しているもの、 参考 は設計判断の裏付けとして参照したものです。 本製品は検出できた範囲のみを証跡付きで報告し、欠陥が無いことの証明はしません。標準への準拠を主張するものではなく、観点の網羅の拠り所として用いています。

標準・規格

  • ISO/IEC/IEEE 29119 — ソフトウェアテスト標準 実装に反映

    テスト方針・戦略・計画・設計・実行の文書体系。AutoRun が生成する test_plan.md / test_analysis.md / test_design.md / test_cases.md の構成はこの体系に沿っています。

    softwaretestingstandard.org ↗
  • ISTQB — テスト目的・テストタイプ・テスト設計技法 実装に反映

    同値分割・境界値分析・デシジョンテーブル・状態遷移テストなどの技法体系。実測した入力項目・画面遷移からのテスト設計生成の基礎です。

    istqb.org ↗
  • ISO/IEC 25010 — システム・ソフトウェア製品品質モデル（SQuaRE） 実装に反映

    機能適合性・性能効率・互換性・使用性・信頼性・セキュリティ・保守性・移植性。品質観点マップの分類軸として使用しています。

    iso25000.com / 25010 ↗
  • ISO/IEC 25012 — データ品質モデル 参考

    正確性・完全性・一貫性などのデータ品質特性。データ品質観点の裏付けとして参照。

    iso25000.com / 25012 ↗
  • WCAG 2.2 — Web Content Accessibility Guidelines（W3C） 実装に反映

    アクセシビリティ検査の基準。ただし自動検出できるのは達成基準の一部であり、未検証領域は「未検証」と明記します。

    w3.org / WCAG 2.2 ↗
  • OWASP Top 10 / ASVS 参考

    セキュリティ観点（セッション・CSRF・CORS など）の分類の拠り所。本製品は脆弱性診断ツールではなく、観点の提示にとどめています。

    owasp.org / Top 10 ↗
  • IPA 非機能要求グレード 参考

    情報処理推進機構（IPA）による非機能要求の体系。非機能観点の粒度設計に参照。

    ipa.go.jp ↗
  • ISO 9241-11 — 使用性（ユーザビリティ）の定義 参考

    有効さ・効率・満足度としての使用性の定義。ユーザビリティ観点の枠組みとして参照。

先行研究

  • P. Ammann, J. Offutt — Introduction to Software Testing（2nd ed., 2016） 実装に反映

    プライムパス網羅をはじめとするグラフ網羅基準。画面遷移モデル上のパス選定（全画面／全遷移／プライムパス）の理論的裏付けです。

  • D. R. Kuhn, R. N. Kacker, Y. Lei — 組合せ（ペアワイズ）テスト 実装に反映

    NIST による組合せテストの実務研究。入力項目の組合せ爆発を抑えるペアワイズ生成に反映。

  • T. Y. Chen ほか — メタモルフィックテスティング 実装に反映

    期待結果が確定できない場合に、入力間の関係性（メタモルフィック関係）で検証する手法。オラクル問題への対処として候補生成に利用。

  • G. Antoniol ほか — Recovering Traceability Links between Code and Documentation（IEEE TSE, 2002） 実装に反映

    情報検索（TF-IDF）による成果物間トレーサビリティ復元。要件文書と実測画面の突合候補の提示に反映。

  • T. Walsh, P. McMinn, G. Kapfhammer — ReDeCheck: レイアウト故障の自動検出（ISSTA, 2017） 実装に反映

    要素の重なり・はみ出しなどレイアウト故障の検出手法。表示崩れ検知の判定ロジックの基礎です。

  • Q. Luo, F. Hariri, L. Eloussi, D. Marinov — An Empirical Analysis of Flaky Tests（FSE, 2014） 実装に反映

    不安定テスト（flaky）の原因分類。非同期待ち（async wait）に起因する失敗の分類・提示に反映。

  • A. Stocco, M. Leotta, F. Ricca, P. Tonella — APOGEN: Page Object の自動生成 実装に反映

    Web テストの保守性を高める Page Object パターンの自動生成。生成スクリプトの構造化に反映。

  • J. Grigera, A. Garrido, J. M. Rivero, G. Rossi — ユーザビリティ・スメルの自動検出 実装に反映

    操作ログ・UI 構造から使いにくさの兆候（usability smell）を検出する研究。ユーザビリティ観点の自動検出に反映。

  • B. Eriksson, G. Pellegrino, A. Sabelfeld — Black Widow: Blackbox Data-driven Web Scanning（IEEE S&P, 2021） 実装に反映

    画面をまたぐデータフローを辿る深いクロール手法。データフロー追跡とフォーム到達の設計に反映。

  • LLM によるWebテスト自動生成の動向 参考

    画面グラフ探索＋LLM によるテスト生成は近年急速にコモディティ化しています。本製品の差別化は文書駆動の入力・証跡と主張範囲の明示・ドリフト検知にあると位置づけています。
