ユーザーストーリー
ペルソナ A：QA エンジニア（主ユーザー）
第三者検証会社から業務委託でアジャイル現場に参画。ドキュメントがない状態で、外部結合テスト・システムテストを担当。テスト設計を早期に開始したい。
ストーリー優先度
URL を入力するだけで、テスト対象の全画面一覧を自動取得したいMust
画面遷移図を自動生成し、結合テストのパスを設計したいMust
各画面の入力フォーム・フィールドを一覧化し、境界値テストの対象を把握したいMust
AI が推定した各画面のテスト観点を参考にしたいShould
生成した文書を Excel でエクスポートし、テスト管理ツールに転記したいShould
ペルソナ B：プロジェクトマネージャー
検証プロジェクト全体を管理。クライアントへの進捗報告・成果物提出が必要。
ストーリー優先度
生成された文書をクライアントに提出できる品質の HTML レポートにしたいShould
プロジェクト開始初日に対象システムの構造を把握したいMust
ペルソナ C：クライアント（開発現場）
アジャイル開発を行う自社開発チーム。第三者検証チームに QA を依頼している。
ストーリー優先度
ドキュメントを整備する工数をかけずに、第三者検証を受けたいCould
生成された文書が自社のドキュメント整備にも役立つとよいCould

機能要件
クローリング機能
ID要件優先度備考
FR-001URL を入力として受け取り、Playwright でページを取得するMustSPA 対応（JS レンダリング後）
FR-002内部リンクを再帰的に追跡し、指定深さまでクロールするMustデフォルト depth=3
FR-003外部ドメインへのリンクは追跡しないMustホスト名で判定
FR-004最大ページ数を制限できるMustデフォルト max=50
FR-005各ページのスクリーンショットを保存するShouldPNG 形式
FR-006ネットワークアイドル状態を待機して動的コンテンツを確保するMustnetworkidle 使用
解析機能
ID要件優先度備考
FR-007ページの URL・タイトル・h1〜h3 見出しを抽出するMust
FR-008フォーム・input・select・textarea 要素を抽出するMusttype, name, placeholder 含む
FR-009ボタン・リンクのテキストとリンク先を抽出するMust
FR-010同一 URL の重複を排除するMust正規化後に判定
文書生成機能
ID要件優先度備考
FR-011画面一覧表を Markdown テーブルで出力するMustURL/タイトル/主要要素/遷移先
FR-012画面遷移図を Mermaid 形式で出力するMustgraph LR 形式
FR-013入力項目一覧を Markdown テーブルで出力するMustページ別・フィールド属性含む
FR-014Excel 形式（.xlsx）でシート別に出力するShouldopenpyxl 使用
FR-015HTML レポートを生成するShouldスクショ埋め込み
LLM 解析機能（オプション）
ID要件優先度備考
FR-016Claude API（Haiku）を用いて各画面の目的・役割を推定するCould--llm フラグで有効化
FR-017ユーザーが実行できる操作一覧を AI で推定するCould
FR-018入力フィールドの推定バリデーションルールを出力するCould
FR-019QA テスト観点を 3 点推定して出力するCould
FR-020処理済みページはキャッシュして再処理しないShouldコスト削減

非機能要件
カテゴリ要件目標値
性能50 画面のクロール・文書生成時間（LLM なし）5 分以内
性能50 画面のクロール・文書生成時間（LLM あり）15 分以内
信頼性クロール中のタイムアウトエラー時の継続動作エラーをスキップして継続
セキュリティAPI キーの管理.env ファイル / コードにハードコード禁止
セキュリティ出力ファイルの管理output/ は .gitignore 必須
保守性ファイルサイズ1 ファイル 800 行以内
保守性テストカバレッジ80% 以上
移植性動作環境Python 3.12 / macOS・Linux
使いやすさCLI インターフェース--url のみ指定で動作すること
4
制約条件
制約内容理由
対象ページログイン不要の公開ページのみ認証情報を扱うとセキュリティリスクが生じるため（Phase 1 制約）
クロール深さデフォルト depth=3（変更可）処理時間と網羅性のバランス
ページ数上限デフォルト max_pages=50（変更可）大規模サイトでの処理時間超過防止
言語Python 3.12Playwright との整合性（3.13 は greenlet ビルド失敗）
LLMClaude Haiku のみ（Sonnet 以上は使わない）大量ページ処理でのコスト抑制
ライセンス社内利用限定クライアント情報が出力に含まれる可能性

受け入れ基準
Phase 1（MVP）受け入れ基準
#基準確認方法
AC-01python src/main.py --url https://example.com が正常終了する手動実行
AC-02output/ に screens.md が生成されるファイル確認
AC-03screens.md に URL・タイトル・遷移先を含む画面一覧が含まれる内容確認
AC-04output/ に transition.mmd が生成されるファイル確認
AC-05transition.mmd を Mermaid レンダラーで表示すると遷移図が描画されるmermaid.live で確認
AC-0650 画面のクロールが 5 分以内に完了する実行時間計測
AC-07テストカバレッジが 80% 以上であるpytest --cov
Phase 2（LLM）受け入れ基準
#基準確認方法
AC-08--llm フラグ付きで実行すると各画面に「テスト観点」が追記される内容確認
AC-09同一 URL の LLM 解析は 2 回目以降キャッシュから返されるログ確認
AC-10ANTHROPIC_API_KEY が未設定で --llm を使うと分かりやすいエラーが出る手動確認
