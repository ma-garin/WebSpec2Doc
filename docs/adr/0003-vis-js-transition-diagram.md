# 画面遷移図をMermaid.jsからvis.jsに切り替える

## 文脈

クロール結果からMermaid.jsで画面遷移図を生成していた。しかし実際のサイト（28ページ程度）でも、ヘッダー/フッターの共通ナビゲーションリンクが全ページから出るためエッジが爆発し、図が読めない状態になっていた。またMermaidは静的なSVG出力であり、ノードクリックで対応画面の仕様を開くといったインタラクションが実装困難だった。

## 決定

**画面遷移図のレンダリングをMermaid.jsからvis.jsに切り替える。**

- vis.jsのネットワーク図（force-directed layout）を使用する
- ノードクリックで「画面別仕様」タブの対応画面に遷移する
- 共通ナビゲーション除去: 全クロール済みページの50%以上から出発するエッジは共通ナビと判定して非表示にする
- フォーム送信エッジ（`form.action`由来）をリンクエッジと色で区別する

## 棄却した代替案

- **Mermaidのままデータを改善**: 共通ナビ除去でエッジは減るがズーム/パン/クリックが実装困難。密なグラフでのレイアウト品質もvis.jsに劣る。
- **D3.js**: vis.jsより高機能だが実装コストが高い。このユースケースにはvis.jsで十分。

## 帰結

- `src/generator/mermaid_generator.py` と `transition.mmd` の出力は後方互換のため残す（CLI・エクスポート用途）
- GUIの画面遷移図タブはvis.jsで描画するよう `app-view.js` を更新する
- vis.jsはCDNから遅延ロード（Mermaidと同方式）

## 追記（2026-06-03）

GUIの画面遷移図は、QA資料としての読みやすさを優先し、Mermaid UML 3タブ + テスト観点マップへ再整理した。

- シーケンス図: 操作順を代表遷移で表示
- コミュニケーション図: 画面間の全体関係を番号付きエッジで表示
- アクティビティ図: テスト手順として操作と期待結果を表示
- 各タブは図だけで完結させず、図のエッジ番号と対応表のNoを揃える
- テスト観点マップ: 遷移を到達性、入力後遷移、認証・会員導線、業務クリティカル導線へ分類する

vis.jsのインタラクティブ性より、テスト設計時に図と表を照合できることを優先した。
