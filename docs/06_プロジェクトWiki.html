1
全体像
WebSpec2Doc は、URL を入力として Web サイトをクロールし、QA テスト設計に必要な画面一覧・フォーム一覧・画面遷移図・HTML/Excel レポートを生成する Python ツールです。
アプリ種別
CLI+ Web UI
中核技術
Playwright
出力
MDHTML / Excel
現在の位置づけ

実装済みの MVP は「公開ページのクロール」「内部リンク抽出」「フォーム抽出」「遷移グラフ生成」「Markdown / Mermaid / HTML / Excel 出力」「仕様ドリフト検知（--compare / スナップショット差分）」「ブラウザ GUI（Flask）」までを扱う。LLM 連携用のフラグとディレクトリは存在するが、現状の --llm は未実装として無視される。

3
構成と処理フロー
1. 入力
src/main.py が URL、深度、最大ページ数、出力形式を受け取る。
2. クロール
page_crawler.py が Playwright でページを巡回し、robots.txt と深度制限を考慮する。
3. 抽出・解析
link_extractor.py と analyzer がリンク、見出し、フォーム、入力項目を整理する。
4. グラフ化
transition_graph.py が networkx.DiGraph を構築する。
5. 出力
Markdown、Mermaid、HTML レポート、Excel 仕様書を output/{domain}/ に保存する。

テストpytest / pytest-cov / pytest-playwright純粋関数中心にテスト。

運用メモ
操作コマンド / 入口補足
CLI 実行python src/main.py --url https://example.com --format md,html基本の実行方法。
シェル経由./run.sh https://example.com --depth 2venv 有効化込み。
Web UIpython app.py または WebSpec2Doc.commandFlask UI から実行。
依存導入pip install -r requirements.txt必要時のみ実行。Playwright ブラウザ導入も別途必要。
出力確認output/{domain}/Markdown、Mermaid、HTML、Excel、screenshots が生成される。
現在の注意点
  • ログイン必須ページは README 上では対象外。将来ロードマップでは高優先度。
  • --llm は引数として存在するが、現状は未実装警告のみ。
  • output/ は機密情報を含む可能性があるため、Git に含めない方針。
  • HTML レポートは Mermaid CDN を読み込むため、完全オフライン配布時は注意が必要。
