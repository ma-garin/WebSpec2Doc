# WebSpec2Doc ユーザーガイド

URL を渡すだけで QA テストインプット文書を自動生成するツール。

---

## セットアップ（初回のみ）

```bash
# Python 3.12 が必要（3.13 は非対応）
python3.12 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

pip install -r requirements.txt
playwright install chromium
```

---

## GUI で使う（推奨）

```bash
python app.py
# → ブラウザが自動で http://127.0.0.1:8765 を開きます
```

### 4ステップの流れ

#### ステップ 1: URL 入力・解析

1. 「+ サイトを追加」をクリック
2. クロール対象の URL を入力（例: `https://example.com`）
3. **「画面分析」** ボタンを押す
4. 分析完了後、「N 件の画面を検出しました（経過時間）」と表示される
5. **「次へ →」** をクリック

#### ステップ 2: 条件設定

**画面の選択**
- 取得する画面にチェックを入れる（デフォルト：全選択）
- 不要な画面はチェックを外す

**ログインが必要なサイトの場合**
- 「要ログイン」バッジが付いた画面は画面リストの下部にグループ表示される
- 各「要ログイン」画面の直下に、**その画面専用の** ID/PASSWORD 入力フォームが自動表示される
- 検出されたログイン URL（例: `ログインURL: /login`）がフォーム上部に表示される
- **ID**（ユーザーID・メールアドレス）と **PASSWORD** を入力して **「ログイン」** を押す
- 自動ログイン後、認証後ページを含む再解析が実行される
- パスワードはセッション確立後に即破棄（保存されない）
- 複数の認証エリアがある場合は画面ごとに個別入力する

**その他のオプション**
- 「前回との差分を出力」をチェックすると仕様ドリフト検知が有効になる

**「解析開始」** をクリックしてステップ 3 へ進む。

#### ステップ 3: 実行

- クロールの進捗が 4 段階（受付 → ページ取得 → 解析 → 結果整理）でリアルタイム表示される
- ライブプレビューで取得中の画面を確認できる
- 完了すると **「レポートを見る →」** ボタンが表示される（実行時間付き）
- ボタンをクリックしてステップ 4 へ進む

#### ステップ 4: レポート

8 つのタブで成果物を確認できる。

| タブ | 内容 |
|------|------|
| **概要** | 画面インベントリ一覧（画面ID・タイトル・URL・フォーム数・入力項目数・遷移先） |
| **画面別仕様** | スクリーンショット（クリックで全画面表示）＋フォームカード＋テスト条件＋ページ要素（フォームなし画面） |
| **入力項目・テスト条件** | 全フィールドのテスト条件マトリクス（画面/型/必須フィルタ・検索・CSV 書き出し対応） |
| **設計** | テスト設計技法の推奨マトリクス（同値分割/境界値分析/決定表/状態遷移/分類木/PW法 等） |
| **技法詳細** | 各画面・各技法の機械導出サンプル値を確認できる詳細カード |
| **画面遷移図** | UML 遷移図を 3 種類のサブタブで表示：シーケンス図 / コミュニケーション図 / アクティビティ図 |
| **遷移表** | ISTQB 標準フォーマットの画面遷移表（イベント列にリンク先 URL パスを表示） |
| **履歴・差分** | クロール履歴と任意 2 時点の仕様ドリフト比較 |

**成果物のプレビュー** は各成果物カードの **「プレビュー」** ボタンで行う。アプリ内モーダルで表示されるため別ウィンドウは開かない。モーダルは「✕ 閉じる」ボタン・背景クリック・Escape キーで閉じる。別タブで開きたい場合は「↗ 新規タブ」をクリックする。

**エクスポート** はヘッダーの「↓ エクスポート」ボタンからいつでも行える。

| 形式 | 内容 |
|------|------|
| HTML レポート | 画面別仕様・テスト条件・スクリーンショット付き |
| PDF | HTML レポートの印刷・配布用 PDF 版 |
| JSON | 全データの構造化 JSON（自動化・CI 連携用） |
| Excel | spec.xlsx（テスト管理表への転記用） |
| Markdown（画面一覧） | screens.md |
| Markdown（フォーム） | forms.md |
| 遷移図（Mermaid） | transition.mmd |
| 差分レポート | diff_report.html |
| ZIP | 生成物一式をまとめてダウンロード |

---

## 再クロール（仕様ドリフト検知）

ダッシュボードの **「再クロール」** ボタンを押すと、前回の取得対象・設定を自動復元してステップ 2 から再開する。

- 設定を確認・変更してから「解析開始」
- 完了後のレポートの「履歴・差分」タブで前回との差分（追加/削除された画面・変更されたフォーム）を確認できる

## 品質観点を確認する

左サイドメニューの **「品質観点」** から、解析済みサイトの `report.json` を入力にして、CSV観点、アクセシビリティ、OWASP WSTG/ASVS、境界値、ペアワイズ、API契約、ビジュアル回帰候補を確認できる。

標準では外部 LLM API は呼び出さない。既存の `report.json` とQA観点CSVだけで生成する。QA観点には `data/qa_viewpoints_summary.csv` を使い、別CSVを使う場合は `QA_VIEWPOINTS_CSV` 環境変数で差し替えできる。

### 操作方法

1. 左サイドメニューの **「品質観点」** を開く
2. **対象サイト** を選択する
3. **「成果物を生成」** を押す
4. 左ペインの **成果物** から **「プレビュー」** ボタンで画面内確認、またはダウンロードを行う

### 生成される品質観点成果物

`output/{ドメイン名}/qa_process/` に保存される。

| ファイル | 内容 |
|---------|------|
| `quality_viewpoints.json` | 品質観点データ |
| `quality_viewpoints.html` | 品質観点HTMLレポート |

### トレーサビリティID

テスト設計とテストケースには、元画面・元フォーム・元入力項目へ戻れる Trace ID が付く。

| 例 | 意味 |
|----|------|
| `P001` | 画面ID |
| `P001-F01` | P001 の1つ目のフォーム |
| `P001-F01-I01` | P001 の1つ目のフォームの1つ目の入力項目 |
| `P001->P002` | P001 から P002 への画面遷移 |

## CLI で使う

```bash
# 最小構成
python src/main.py --url https://example.com

# HTML レポート付き（推奨）
python src/main.py --url https://example.com --format md,html,excel,json

# 仕様ドリフト検知
python src/main.py --url https://example.com --compare

# 認証が必要なサイト（手動セッション保存）
python src/main.py --login https://example.com/login   # ブラウザでログイン後 Enter
python src/main.py --url https://example.com --auth auth.json
```

### オプション一覧

| オプション | デフォルト | 説明 |
|-----------|-----------|------|
| `--url` | 必須 | クロール対象 URL |
| `--depth` | `3` | リンクを辿る深さ |
| `--max-pages` | `50` | クロールする最大ページ数 |
| `--output` | `./output` | 出力先ディレクトリ |
| `--format` | `md` | 出力形式（`md,html,excel,pdf,json` をカンマ区切り） |
| `--compare` | off | 前回スナップショットとの差分を出力 |
| `--auth` | — | 保存済みセッション（auth.json）を指定 |

## 出力ファイル

実行後に `output/{ドメイン名}/` に生成される。

| ファイル | 内容 |
|---------|------|
| `report.html` | 画面別仕様・テスト条件・スクリーンショット付きレポート |
| `report.json` | 全データの構造化 JSON |
| `spec.xlsx` | Excel 仕様書 |
| `screens.md` | 画面一覧（Markdown） |
| `forms.md` | フォーム・入力項目一覧（Markdown） |
| `transition.mmd` | 画面遷移図（Mermaid 形式） |
| `report.pdf` | PDF 版レポート |
| `diff_report.html` | 仕様ドリフト差分レポート |
| `screenshots/` | 各画面のスクリーンショット（PNG） |
| `snapshots/` | ドリフト検知用スナップショット |
| `qa_process/` | 品質観点などの補助成果物 |

## よくある質問

**Q: ログインが必要なページが取得できない**
→ ステップ 1 で「画面分析」を実行すると、認証が必要な画面は「要ログイン」バッジ付きで画面リスト下部にグループ表示されます。各画面の直下に表示される専用フォームに ID/PASSWORD を入力して「ログイン」を押してください。複数の認証エリアがある場合は画面ごとに個別入力します。

**Q: 画面遷移図がごちゃごちゃしている**
→ 共通ナビゲーション（ヘッダー/フッター）のリンクは自動的に除外されます。「画面遷移図」タブのサブタブ（シーケンス図 / コミュニケーション図 / アクティビティ図）を切り替えると、同じ遷移情報を別の視点で確認できます。

**Q: テスト設計技法の詳細なサンプル値を確認したい**
→ 「技法詳細」タブで、各画面・各技法の機械導出サンプル値（境界値・同値クラス・組合せ等）を確認できます。

**Q: 品質観点は追加料金がかかる？**
→ 標準のローカル生成ではかかりません。外部LLM/APIは呼び出しません。

**Q: 品質観点画面に対象サイトが出ない**
→ 先に「+ サイトを追加」からサイトを解析し、`report.json` を生成してください。旧データで `report.json` がない場合は再クロールが必要です。

**Q: 成果物を確認するたびに別ウィンドウが開く**
→ 各成果物カードの「プレビュー」ボタンを使うとアプリ内モーダルで表示されます（別ウィンドウは開きません）。モーダルは「✕ 閉じる」・背景クリック・Escape で閉じます。あえて別タブで開きたい場合は「↗ 新規タブ」をクリックしてください。

**Q: 品質観点で成果物が生成後に見つけにくい**
→ 品質観点画面は2ペインレイアウトで、左サイドバーに成果物リストが常時表示されます。生成後すぐ左サイドバーの成果物名をクリックするとプレビューできます。

**Q: 前回との差分を確認したい**
→ ダッシュボードから「再クロール」→ 「前回との差分を出力」にチェック → 「解析開始」。レポートの「履歴・差分」タブで確認できます。

**Q: `greenlet` のビルドが失敗する**
→ Python 3.13 は非対応です。`python3.12 -m venv venv` で Python 3.12 の仮想環境を作成してください。

**Q: P1 の画面分析がタイムアウトする**
→ 画面分析（discover）のタイムアウトは 180 秒です。ネットワークが遅い環境や大規模サイトでは超過する場合があります。対象 URL を絞るか、ステップ 2 の手動 URL 入力モードを使用してください。
