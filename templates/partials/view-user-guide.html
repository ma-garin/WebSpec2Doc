WebSpec2Doc ユーザーガイド

URL解析からレポート確認、品質観点レビューまで、画面上で確認できる操作ガイドです。

1. サイトを追加左上の「サイトを追加」から対象URLを入力します。
2. 画面分析「画面分析」で取得対象ページを検出します。
3. 条件設定対象画面、ログイン、差分出力を確認します。
4. レポート確認概要、画面別仕様、入力項目、遷移、履歴差分を確認します。
5. 品質観点CSV、a11y、OWASP、境界値、API契約の観点を確認します。

CLIの使い方（画面を使わずコマンドで実行する）

画面と同じ処理は python src/main.py からコマンドラインでも実行できます。バッチ処理・自動化・CI連携に使います。

オプション役割例
--url解析対象のURL（クロールの起点）--url https://example.com/
--format出力形式（指定可能: md,html,excel,pdf,json。既定は md のみ）。カンマ区切りで複数指定可--format html,pdf,excel,json,md
--compare前回のクロール結果（スナップショット）と比較し、仕様ドリフトを検出する--compare
--fail-on-driftドリフト検出時に exit code 1 で終了する（CIでの失敗判定に使う）--compare --fail-on-drift
--login-record見えるブラウザでログイン操作を記録し、認証セッション（auth.json）を保存する。--login-record-url（ログインページ）と --login-signal（完了シグナルファイル）を併用する--login-record --login-record-url https://example.com/login --login-signal .signal
--auth保存済みの認証セッションファイル（auth.json）を指定してクロールする--auth auth.json
--depthリンクをたどる階層数（既定 2）--depth 3
--max-pages取得する最大ページ数（既定 30）--max-pages 50

代表的な使い方の流れ:

  1. 初回クロール（仕様書一式を生成）: python src/main.py --url https://example.com/ --format html,pdf,excel,json,md
  2. ログインが必要なサイトは、画面の「ログイン情報の設定」またはCLIの --login-record で先にセッションを記録する
  3. 記録した認証情報でクロール: python src/main.py --url https://example.com/mypage --auth auth.json
  4. リリース後の再クロールで仕様ドリフトを検知: python src/main.py --url https://example.com/ --compare
  5. CIパイプラインでドリフトを失敗判定にする: python src/main.py --url https://example.com/ --compare --fail-on-drift

全オプション一覧は python src/main.py --help で確認できます。ログイン記録CLIの詳細フラグは複雑なため、通常は画面（設定 →「ログイン情報の設定」）からの記録を推奨します。

CI/CD 連携（仕様ドリフトの自動監視）

デプロイのたびに仕様ドリフトを自動検知できます。差分があると exit code 1 で終了するため、パイプラインで検知可能です。

  • CLI: python src/main.py --url https://staging.example.com --compare --fail-on-drift
  • GitHub Actions: リポジトリ同梱の .github/workflows/spec-drift.yml をそのまま利用できます（毎週月曜の定期実行 + 手動実行）。リポジトリ変数 WEBSPEC_TARGET_URL に対象URLを設定してください。
  • Slack通知: 設定 → 通知タブで Incoming Webhook URL を登録すると、ドリフト検知時に自動でSlackへ通知します。
  • 社内環境のクロール: ステージング等のプライベートアドレスを対象にする場合は環境変数 WEBSPEC2DOC_ALLOW_LOCAL=1 を設定してください（既定では SSRF 保護のため拒否されます）。

よくある確認

  • 品質観点に対象サイトが出ない場合は、先にサイトを解析して report.json を生成してください。
  • ログインが必要な画面は、条件設定で認証情報を入力してから解析してください。
  • 前回との差分は「再クロール」後、レポートの「履歴・差分」タブで確認できます。
