設計原則
原則適用方法
イミュータブルデータ全モジュールは入力を変更せず新オブジェクトを返す（dataclass 使用）
疎結合各モジュールは型付きインターフェースのみで連携（import 依存を最小化）
フェイルセーフ1 ページのクロール失敗は警告ログのみ、処理継続
ファイルサイズ制限1 ファイル 800 行以内・1 関数 50 行以内
2
コンポーネント仕様
データモデル（共通型）
@dataclass(frozen=True)
class PageData:
    url: str
    title: str
    headings: tuple[str, ...]      # h1-h3 テキスト
    links: tuple[str, ...]         # 内部リンク URL
    forms: tuple[FormData, ...]    # フォーム情報
    screenshot_path: str | None

@dataclass(frozen=True)
class FormData:
    action: str
    method: str
    fields: tuple[FieldData, ...]

@dataclass(frozen=True)
class FieldData:
    field_type: str    # text, email, password, select...
    name: str
    placeholder: str
    required: bool
page_crawler.py — 主要関数
関数引数戻り値説明
crawl_site()url, depth, max_pages, output_dirlist[PageData]サイト全体をクロールして PageData リストを返す
crawl_page()page, url, output_dirPageData単一ページのデータを取得する
is_internal_link()base_url, link_urlbool同一ドメインの内部リンクか判定する
normalize_url()urlstrURL を正規化してフラグメント・重複パラメータを除去する
transition_graph.py — グラフ構造
要素属性説明
ノード（画面）url, title, page_id, forms_count, fields_count各ページを表す頂点
エッジ（遷移）link_text, element_type（link/button/form）ページ間の遷移を表す有向辺
グラフ型networkx.DiGraph有向グラフ（同一ページ間の多重エッジも保持）
3
データフロー
ステップ入力処理出力
1. クロール 起点 URL（str） Playwright でページを取得。JS 実行後の DOM を解析。 list[PageData]
2. HTML 解析 list[PageData] 各ページの見出し・リンク・フォームを構造化 list[AnalyzedPage]
3. グラフ構築 list[AnalyzedPage] URL 間のリンク関係を有向グラフに変換 networkx.DiGraph
4a. Mermaid 生成 DiGraph ノード・エッジを Mermaid graph LR 形式に変換 transition.mmd（str）
4b. Markdown 生成 list[AnalyzedPage] 画面一覧・入力項目一覧を Markdown テーブルに変換 screens.md（str）
4c. LLM 解析（オプション） AnalyzedPage + screenshot Claude Haiku に HTML 要素を送信し、目的・操作・テスト観点を取得 LLMInsight[]
5. 出力保存 全生成物 output/{domain}/ に各ファイルを保存 ファイル群

外部 API 仕様
Claude API（Anthropic）
項目値
モデルclaude-haiku-4-5-20251001
エンドポイントAnthropic SDK 経由（直接 HTTP 呼び出しは使わない）
最大トークン1,024（応答） / ページあたり約 2,000（入力）
プロンプトキャッシュ有効（システムプロンプトをキャッシュ）
レート制限SDK のデフォルトリトライ（3 回）を使用
プロンプト設計
# system（キャッシュ対象）
あなたは QA エンジニアの支援 AI です。
Web ページの HTML 構造から、テスト設計に役立つ情報を抽出してください。
必ず JSON 形式で回答してください。

# user（ページ毎に変化）
URL: {url}
タイトル: {title}
見出し: {headings}
フォーム: {forms}
ボタン: {buttons}

以下の JSON で回答してください:
{
  "purpose": "画面の目的（1行）",
  "operations": ["操作1", "操作2", ...],
  "test_points": ["テスト観点1", "テスト観点2", "テスト観点3"]
}
Playwright
項目値
ブラウザChromium（playwright install chromium）
モードヘッドレス（headless=True）
待機戦略wait_until="networkidle"（最大 30 秒）
ビューポート1280 × 800
タイムアウトページ読み込み：30 秒 / 要素待機：5 秒
