# WebSpec2Doc 実機デモ台本

ビジネスアイデア大会向けの実演手順。同梱デモサイト **DemoMart** を対象に、
外部ネットワーク・OpenAI API キーなしで完走する。

## 事前準備（発表前に済ませておく）

```bash
make demo
```

- デモサイト（DemoMart）: http://127.0.0.1:8766/
- WebSpec2Doc 本体: http://127.0.0.1:8765/
- ネットワーク遮断環境でも動作（顧客データを外部送信しないことの証明）
- 失敗保険: `docs/demo/sample_output/report.html` を別タブで開いておく。
  ライブ生成が失敗しても、この生成済みレポートで説明を続行する。

## 5分版（ピッチ本番用）

| 時間 | 操作 | 話すこと（要点） |
|---|---|---|
| 0:00 | （スライド1枚） | 「資料のない稼働中システムに、テスト設計者は初日から入れない。この空白を URL 一つで埋めます」 |
| 0:30 | 本体 URL 欄に `http://127.0.0.1:8766/` を入力→画面分析 | 「まず何画面あるかを自動発見。ログインが要る画面も検出します」 |
| 1:00 | クロール実行（ライブプレビュー） | 「robots.txt を尊重し、リクエスト間隔も自動制御。**管理画面は robots で自動スキップ**され監査ログに残ります（コンプライアンス）」 |
| 1:45 | レポート「画面カタログ」→ 決済画面の**根拠バッジ**をクリック | 「各入力項目に根拠（rules＋確信度）が付き、クリックでスクショの該当位置をハイライト。**推測ではなく実測に基づく**ことが売りです」 |
| 2:30 | 「遷移テストカバレッジ」セクション | 「テスト網羅率を **ISO/IEC/IEEE 29119-4 準拠**で算出。出典まで明記します」 |
| 2:50 | 「ビジネスフロー」表 | 「決済画面を通るフローを**自動で優先度『高』**に。リスクの高い所からテストできます」 |
| 3:15 | デモサイトの checkout.html を1箇所改変→再クロール（or sample_output の差分を見せる） | 「仕様が変わると**ドリフト差分**を検出し、**影響を受けるテストと再実行推奨リスト**を提示。Datadog がインフラを監視するように、私たちは『Web 仕様』を監視します」 |
| 4:00 | ROI ダッシュボード（マイページ→利用実績） | 「そして最大のポイント。**削減できた工数を製品自体が記録**します。これが採用後の効果測定レポートになります」 |
| 4:40 | Excel / TestRail エクスポート | 「成果物は Excel・TestRail 形式で既存業務にそのまま接続」 |
| 5:00 | 締め | 「明日から社内案件に投入でき、6ヶ月で黒字化できます」 |

## 10分版（質疑込み・詳細デモ）

5分版に加えて、以下を挟む:

1. **認証ウォール検出**（1:15 付近）: ログイン画面が「要ログイン」として検出される様子。「認証の必要な画面も、ログインすれば認証後ページまで解析します」
2. **画面状態の検出**（2:00 付近）: マイページのモーダル・タブ・アコーディオンが「別の画面状態」として記録されること。「同じ URL でも状態が違えば別画面として扱う=SPA・動的サイトに対応」
3. **バリデーション実測**（2:20 付近）: 必須フォームを空送信したときのブラウザ実測メッセージが「期待結果（実測）」に転記される様子。「テスト条件の期待結果を、人手ではなく実測で埋めます」
   - 注: メッセージ言語はブラウザのロケールに従う（日本語 OS では日本語で表示）
4. **SPA 遷移捕捉**（3:00 付近）: キャンペーンページの pushState/hashchange が遷移グラフに反映される様子

## デモの見せ場と対応機能（早見表）

| 見せ場 | デモサイトのページ | 裏で動く機能 |
|---|---|---|
| robots 自動スキップ・監査ログ | /admin.html（robots で拒否） | クロール礼儀・audit.jsonl |
| 根拠バッジ・bbox ハイライト | /checkout.html | SourceEvidence / confidence |
| 認証ウォール検出 | /login.html | login_wall 検出 |
| 決済フロー優先度「高」 | /checkout.html（SCREEN_PAYMENT） | ビジネスフロー優先度付け |
| PII リスク検出 | /checkout.html | pii_risk_screens |
| 画面状態（モーダル/タブ/アコーディオン） | /dashboard.html | 状態ベース探索 |
| SPA 遷移 | /spa.html | pushState/hashchange 捕捉 |
| バリデーション実測 | /contact.html, /checkout.html | dry-run 実測 |
| 29119-4 カバレッジ | サイト全体 | N-switch カバレッジ |
| ドリフト差分＋影響テスト | 再クロール時 | impact_analyzer |
| ROI 実測 | 本体マイページ | usage_tracker（Week2 実装） |

## トラブル時の切り替え

- ライブクロールが遅い／失敗 → `docs/demo/sample_output/report.html` を開いて成果物ベースで説明
- ブラウザが自動で開かない → 手動で http://127.0.0.1:8765/ を開く
- ポート衝突 → `DEMO_SITE_PORT=8890 make demo`
