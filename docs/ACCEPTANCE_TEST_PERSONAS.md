## 全体観察（評価前提）

### 実測した構造的バグ（C-1）
`report.json` 集計より：28画面のうちユニークなパスは **16**。`/ja/reserve.html` が7回・`/en-US/reserve.html` が7回、計**12画面が同一フォームの `?plan-id=N` クエリ違いで別画面計上**。エグゼクティブサマリーの「**808ケース/135時間**」「リスク上位3画面（P007/P008/P009）」はすべてこの重複に汚染されている。**この数値を信じて顧客提案に使うと恥をかく。**

### 追加観察（実機）
- 品質観点・自動テスト候補・モデル/カバレッジは「サイト選択→生成ボタン」まで**初期表示が完全に空**
- 設定モデルタブに「LLMテスト観点生成（**今後実装**）で使うモデル」と明記
- ユーザーガイドに CI/CD (`--fail-on-drift`) ・Slack通知・`WEBSPEC2DOC_ALLOW_LOCAL=1` の説明あり。**GUI への露出が弱い**
- 出力は report.html / report.json / spec.xlsx / report.pdf / screens.md / QAプロセスファイル群 — 形式は多様で **ツール依存しない**（TestRail/Jira等は利用者の環境次第で各自連携できる）
