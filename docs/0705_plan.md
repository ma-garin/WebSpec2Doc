## 検証済み根本原因（P0）

### P0-1: AutoRun 全テスト失敗

実測: `output/.playwright_env/package.json` = `@playwright/test ^1.56.1`、Python側 `playwright==1.44.0`。`_ensure_pw_env`（web/services/playwright_executor.py:338-369）が `npx playwright --version`（環境の最新）でバージョン決定→不一致。1.56は `chromium_headless_shell-1217` を要求、`.runtime/ms-playwright` には1.44用chromiumのみ。

**修正**:
- (a) `playwright.__version__` にピン止め（package.jsonにdependencies記録）
- (b) 既存envのバージョン不一致検出→再構築
- (c) 実行前ブラウザ実在検証・無ければ `npx playwright install chromium`（PLAYWRIGHT_BROWSERS_PATH明示）で自動修復
- (d) failure_classifier に「実行環境エラー」パターン追加（browserType.launch/Executable doesn't exist）→ 334件未分類ではなく1件の環境エラー＋修復手順として表示

### P0-2: 認証情報入力パネルの潰れ（画像1）

実測: パネル本体レイアウトが100%インラインstyle依存（`.disc-item-login-body` のCSSは**空ルール** static/app-report.css:271）、外枠 `.disc-login-item-wrap` が `overflow:hidden` → 高さ喪失時にオレンジ縞に潰れる構造。

**修正**: パネルにbase CSS（display/flex/min-height）を付与しインライン依存を解消、`overflow:hidden` を見直し、描画高をアサートするE2E追加。
