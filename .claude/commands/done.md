# /done — 完了ゲート確認

引数: $ARGUMENTS（対象タスク、PR、または空）

## 実行内容

done-gateスキルに従い、作業を完了扱いにできるか確認します。

1. 対象タスクと変更範囲を確認する
2. qa-review-standardsの観点でCritical/Highが残っていないか確認する
3. test-automationの基準で必要なテストが通っているか確認する
4. 秘密情報、未更新ドキュメント、未コミット変更、検証証跡の不足を確認する
5. 未達項目があれば「未完了」として残り作業を列挙する

全項目を満たす場合のみ「完了基準を満たしている」と判定する。
