## 1. このツールについて

WebSpec2Doc は「URL を渡すだけで QA テストインプット文書を自動生成する」Python ツールです。

### 解決する課題

アジャイル開発・DevOps 現場での第三者検証支援において、こんな状況がよくあります。

> 「ドキュメントがないので、まず触って覚えてください」

これでは**テスト設計の網羅性が担保できません**。WebSpec2Doc はウェブサイトを自動でクロールし、以下を生成することでこの問題を解決します。

- **画面一覧** — どんな画面が存在するかを一覧化
- **画面遷移図** — 画面間の移動経路を可視化
- **入力項目仕様** — フォームのフィールド・型・必須可否を自動抽出
- **スクリーンショット付きレポート** — 視覚的に確認できる HTML 文書

### 対象サイト

- 公開されているウェブサイト・ウェブアプリケーション
- JavaScript でレンダリングされる SPA（シングルページアプリ）にも対応
- **ログインが必要なページは対象外**（認証ページまでは到達可能）

## 2. 動作環境

| 項目 | 要件 |
|---|---|
| OS | macOS / Linux / Windows |
| Python | 3.12（3.13 は greenlet ビルド失敗のため非対応） |
| ネットワーク | クロール対象サイトへの HTTP/HTTPS アクセス |

## 3. インストール

### 手順

```bash
# リポジトリをクローン（または ZIP 展開）後、プロジェクトディレクトリへ移動
cd 008_CrateSpec2HTML

# 仮想環境を作成・有効化（Python は 3.12 を使う）
python3.12 -m venv venv
source venv/bin/activate        # macOS / Linux
# venv\Scripts\activate         # Windows

# 依存パッケージをインストール
pip install -r requirements.txt

# Playwright ブラウザ（Chromium）をインストール
playwright install chromium
```

### 動作確認

```bash
python src/main.py --help
```

以下のような出力が表示されれば成功です。

```
usage: main.py [-h] --url URL [--depth DEPTH] ...
```

## 4. 基本的な使い方

### 最小構成

```bash
python src/main.py --url https://example.com
```

`output/example.com/` に以下が生成されます。
- `screens.md` — 画面一覧
- `forms.md` — フォーム一覧
- `transition.mmd` — 遷移図（Mermaid 形式）
- `screenshots/` — 各画面のスクリーンショット

### HTML レポートも生成する（推奨）

```bash
python src/main.py --url https://example.com --format md,html
```

`report.html` が追加生成されます。スクリーンショット・遷移図・テーブルが 1 ファイルにまとまり、そのまま共有できます。

### 実運用での推奨コマンド

```bash
python src/main.py \
  --url https://対象サイト.example.com \
  --depth 2 \
  --max-pages 30 \
  --format md,html
```

## 5. オプション詳細

### `--url`（必須）

クロールを開始する URL を指定します。

```bash
--url https://example.com
--url https://example.com/app/login
```

- スキームは `https://` または `http://` を含めてください
- サブドメインは別サイトとして扱われます（クロール対象外）

### `--depth`（デフォルト: 3）

リンクを辿る深さを指定します。

| depth | 意味 | 目安ページ数 |
|---|---|---|
| `1` | 起点 URL のみ | 1〜数ページ |
| `2` | リンク先まで | 10〜30 ページ |
| `3` | さらにその先まで | 30〜100 ページ |

**推奨**: 初回は `--depth 2` から始め、結果を見て調整してください。

### `--max-pages`（デフォルト: 50）

クロールするページ数の上限です。大規模サイトで実行時間が長くなりすぎる場合に制限します。

```bash
--max-pages 20   # 小規模サイト・素早く確認したいとき
--max-pages 100  # 大規模サイト（処理時間が増加）
```

### `--output`（デフォルト: `./output`）

出力先のベースディレクトリを指定します。実際には `{output}/{ドメイン名}/` に生成されます。

```bash
--output ./output          # デフォルト
--output /tmp/qa-docs      # 絶対パスも指定可能
```

### `--format`（デフォルト: `md`）

生成する出力形式をカンマ区切りで指定します。

| 形式 | 生成ファイル | 用途 |
|---|---|---|
| `md` | `screens.md`, `forms.md`, `transition.mmd` | Notion / Confluence 貼り付け |
| `html` | `report.html` | クライアント提出・ブラウザ閲覧 |
| `excel` | `spec.xlsx` | TestRail / Jira 転記 |

```bash
--format md           # Markdown のみ（高速）
--format md,html      # Markdown + HTML レポート（推奨）
--format md,html,excel  # 全形式
```

### `--compare`（デフォルト: off）

前回のクロール結果（スナップショット）と今回を比較し、**仕様ドリフト差分レポート**（`diff_report.html`）を生成します。

```bash
python src/main.py --url https://example.com --compare
```

- 初回実行時はスナップショットを保存するだけ（差分レポートは次回から）
- 検出する変更：新規画面 / 削除画面 / フォーム（入力項目）の変更 / リンクの変更 / タイトル変更
- スナップショットは `output/{ドメイン名}/snapshots/` にタイムスタンプ名で蓄積されます
- 用途：リリース前後で画面仕様が変わった箇所を特定し、テスト範囲を絞り込む

## 6. 出力ファイルの見方

### `screens.md` — 画面一覧

Markdown テーブル形式で全画面を一覧化します。

```
| # | 画面ID | URL | タイトル | フォーム数 | 遷移先 |
|---|--------|-----|---------|----------|--------|
| 1 | P001 | / | ホーム | 1 | P002, P003 |
| 2 | P002 | /about | 会社概要 | 0 | P001 |
```

- **画面ID**: P001 から連番で付与。テスト仕様書で画面を特定するキーとして使用できます
- **遷移先**: その画面からリンクされている他の画面の ID

### `forms.md` — フォーム・入力項目一覧

各画面のフォームフィールドを一覧化します。

```
| 画面ID | フィールド名 | 型 | 必須 | placeholder |
|--------|------------|-----|------|-------------|
| P001 | email | email | Yes | メールアドレス |
| P001 | password | password | Yes | - |
```

- **型**: `text` / `email` / `password` / `select` / `textarea` など
- **必須**: `required` 属性の有無を検出

### `transition.mmd` — 画面遷移図（Mermaid 形式）

[Mermaid Live Editor](https://mermaid.live/) に貼り付けると遷移図として描画できます。

```
graph LR
  P001["P001<br/>/"]
  P002["P002<br/>/about"]
  P001 -->|リンク| P002
```

### `report.html` — 統合レポート

上記すべてを 1 ファイルにまとめた HTML レポートです。

- ブラウザで開くとスクリーンショット付きで確認できます
- ファイル 1 つをメールやチャットで共有できます
- Mermaid 遷移図はブラウザ上でレンダリングされます（インターネット接続必要）

### `screenshots/` — スクリーンショット

各画面を `P001.png` 〜 `P{n}.png` という名前で保存します。

### `diff_report.html` — 仕様ドリフト差分レポート（`--compare` 指定時）

前回クロールからの変更点を HTML で一覧化します。

- **新規画面 / 削除画面**: 画面の増減
- **フォーム変更**: 入力項目の追加・削除・型/必須/placeholder の変更
- **リンク変更 / タイトル変更**: 遷移経路・画面名の変化

リリース前後で実行し、テストすべき変更箇所を素早く特定するのに使います。

### `spec.xlsx` — Excel 仕様書（`--format excel` 指定時）

| シート | 内容 |
|---|---|
| Screens | 画面一覧（画面ID / URL / タイトル / フォーム数） |
| Forms | 入力項目一覧（画面ID / URL / フィールド名 / 型 / 必須 / placeholder） |

## 7. QA テスト設計への活用方法

### テスト設計の流れ

```
1. WebSpec2Doc でドキュメント生成（5〜10分）
       ↓
2. screens.md で画面一覧を把握
       ↓
3. transition.mmd で遷移経路を確認
       ↓
4. forms.md で入力項目・バリデーション条件を整理
       ↓
5. テスト設計（境界値・同値分割・シナリオテスト）
```

### 画面一覧の活用

- 全画面が網羅されているか確認するチェックリストとして使用
- フォーム数の列を見て、入力テストが必要な画面を素早く特定
- 画面 ID（P001 等）をテスト仕様書の「対象画面」列に記入することで追跡可能に

### 遷移図の活用

- **遷移テスト**: 図に表れているすべての矢印がテストすべき遷移経路
- **ナビゲーション確認**: 意図しないページへの遷移がないか確認
- **異常系**: 正常遷移以外のルートを検討（直接 URL アクセス・戻るボタンなど）

### フォーム仕様の活用

| フィールド情報 | テスト設計への応用 |
|---|---|
| 型（`email`） | バリデーション確認（@ なし・不正形式） |
| 型（`text`） | 最大長・特殊文字・SQL インジェクション |
| 必須（`Yes`） | 空白送信テスト |
| 必須（`No`） | 任意フィールドが空でも送信できるか |
| placeholder | ヒントテキストが消えるか・アクセシビリティ |

### スクリーンショットの活用

- テスト実施前の「期待画面」として参照
- テスト報告書に添付する「エビデンス画面」と比較
- 環境差異（ブラウザ・解像度）の確認

## 8. トラブルシューティング

### SSL 証明書エラーが表示される

```
WARNING: robots.txt を取得できませんでした: [SSL: CERTIFICATE_VERIFY_FAILED]
```

**原因**: macOS の Python が SSL 証明書を参照できていません。  
**影響**: robots.txt の読み込みに失敗するのみ。クロール自体は継続します。  
**解決策**:

```bash
# Python の証明書をインストール
/Applications/Python\ 3.x/Install\ Certificates.command
# または
pip install certifi
```

### ページがクロールされない・少ない

- `--depth` を増やしてみてください（`--depth 2` → `--depth 3`）
- ログイン不要なページのみクロール対象です
- JavaScript でリンクが生成されるサイトは取得できない場合があります

### タイムアウトエラーが頻発する

- 重いページや低速なサーバーの場合に発生します
- `--max-pages` を減らして試してください

### 遷移図（Mermaid）が表示されない

- `report.html` を開く際にインターネット接続が必要です（Mermaid CDN を使用）
- オフライン環境では `transition.mmd` を [Mermaid Live Editor](https://mermaid.live/) に貼り付けてください

### Excel ファイルが開けない

- `openpyxl` がインストールされているか確認してください
- `pip install openpyxl` で解決します
