## 1. 現在地の整理と勝機

### プロダクトの本質的価値

WebSpec2Doc は「稼働中の Web システムの URL を渡すだけで、QA テスト設計に必要な全素材を自動生成する」唯一のツールである。

```
URL入力
  └─ 仕様書生成（HTML/PDF/Excel/JSON）
  └─ テスト観点・テストケース設計
  └─ Playwright 自動化スクリプト
  └─ テスト実行 → 実行レポート
  └─ 仕様ドリフト検知（再クロール差分）
```

### 競合の穴（誰も解いていない問題）

| ツール | 得意なこと | 解決できないこと |
|---|---|---|
| Testim / Mabl | テスト実行・AI修復 | 仕様書・テスト観点の生成 |
| TestRail | テスト管理 | 仕様抽出機能なし |
| Playwright | スクリプト実行基盤 | テスト設計支援なし |
| BrowserStack | クロスブラウザ実行 | 上流設計なし |

**最大の差別化軸**: テスト設計の上流（テストベース生成）に特化した SaaS はグローバルでも存在しない。

### 市場背景

- テストエンジニアの工数の **50〜60%** がテスト設計・仕様理解に費やされる
- アジャイル開発現場の **70%以上** で「仕様書が存在しない or 更新されていない」
- 日本の第三者検証業界では「資料なし現場」が常態化しており、初日から設計を始めるニーズが極めて強い

## 3. 収益モデル設計

### プライシング（月額）

#### Free — ¥0

- 個人利用・3 プロジェクト・月 5 クロール
- ウォーターマーク付き HTML 出力のみ
- **目的**: PLG の入口。QA エンジニアに直接届ける

#### Starter — ¥9,800/月（~$65）

- 個人・無制限クロール・全フォーマット出力
- PDF / Excel / Playwright 候補エクスポート
- **対象**: フリーランス QA・小規模チーム

#### Team — ¥39,800/月（~$270）

- 5 ユーザー・複数プロジェクト管理
- Slack 通知・Jira 連携・AutoRun 完全版
- CI/CD（GitHub Actions / GitLab）
- **対象**: QA 専門会社・スタートアップ

#### Business — ¥119,800/月（~$800）

- 無制限ユーザー・SSO・監査ログ
- ホワイトラベルレポート・API アクセス
- 顧客向けレポート共有機能
- **対象**: 中堅検証会社・SIer の QA 部門

#### Enterprise — 応相談（¥500K~/月）

- オンプレ対応・SLA・専任 CSM
- SOC2・ISMS 対応・カスタム連携
- **対象**: SHIFT Inc.、大手 SIer、金融機関

### 追加収益レバー

| レバー | 内容 | 単価感 |
|---|---|---|
| Usage-based | クロール回数・ページ数超過課金 | ¥50/クロール〜 |
| LLM テスト観点 | AI 生成テストケース（従量） | ¥5/ケース |
| プロフェッショナルサービス | 導入支援・カスタム連携 | ¥500K〜/件 |
| 認定パートナー | 検証会社への再販ライセンス | MRR 20%バック |

## 5. 商用化必須ロードマップ

### 🔴 Phase 1: SaaS 基盤（2〜3 ヶ月・最優先）

**なぜ最優先か**: ローカル実行のままでは導入コストが高く失注する。  
「URL 入力 → 即使える」を実現することが全ての前提。

実装項目:
- ユーザー認証（Auth0 または Firebase Auth / Google・GitHub OAuth）
- 組織・プロジェクト管理（マルチテナント設計）
- Stripe 課金統合（サブスクリプション + Usage-based）
- クロールのバックグラウンドワーカー化（Celery + Redis）
- クラウドデプロイ（AWS Fargate または Render.com）
- 使用量メータリング（クロール数・ページ数・エクスポート数）

### 🔴 Phase 2: チーム機能（2 ヶ月）

**商用価値**: 「個人ツール」から「チームツール」になることでチャーン激減・NRR 向上。

実装項目:
- プロジェクト共有・権限管理（Owner / Editor / Viewer）
- レポートへのコメント・メモ機能
- Slack / Teams 通知（クロール完了・ドリフト検知時）
- 顧客向けレポート URL 共有（認証付き外部共有リンク）

### 🟡 Phase 3: CI/CD インテグレーション（2 ヶ月）

**商用価値**: DevOps ワークフローに組み込まれると解約しにくくなる（スイッチングコスト）。

```yaml
# .github/workflows/qa-drift.yml
- uses: webspec2doc/action@v1
  with:
    url: ${{ secrets.STAGING_URL }}
    format: html,json
    compare: true
    notify_slack: ${{ secrets.SLACK_WEBHOOK }}
```

実装項目:
- GitHub Actions 公式アクション
- GitLab CI テンプレート
- Webhook API（外部トリガー対応）
- REST API（プログラム的アクセス）

### 🟡 Phase 4: 既存ツール連携（2 ヶ月）

| 連携先 | 機能 | 効果 |
|---|---|---|
| Jira | ドリフト検知 → 自動チケット起票 | 購買決裁者（PM）が価値を感じる |
| TestRail | テストケース → TestRail CSV インポート | 乗り換え摩擦ゼロ |
| Notion / Confluence | レポートをページとして同期 | ドキュメント文化の会社に刺さる |
| Slack bot | `/spec check https://...` | 口コミ拡散しやすい |

### 🟢 Phase 5: LLM テスト観点強化（3 ヶ月）

現在: ルールベースでテスト条件を機械導出  
次: LLM（Claude / GPT-4）でテスト観点を自然言語生成

実装項目:
- 「このフォームのリスク観点は？」→ AI 回答
- 画面の業務フローを推論して「見落としリスク」を提示
- セキュリティ観点の自動チェックポイント生成（OWASP 準拠）
- 多言語テストシナリオ（日英自動翻訳）

### 🟢 Phase 6: エンタープライズ機能（3 ヶ月）

- SAML / SCIM SSO（Okta・Azure AD）
- 監査ログ（誰がいつ何をクロールしたか）
- オンプレ / VPC 内デプロイ（Docker Compose / Helm chart）
- SOC2 Type II 取得
- ホワイトラベル（検証会社が自社ブランドで顧客提供）
- SLA 99.9%

## 7. 技術的競合優位（モート）

### モート 1: データネットワーク効果

各クロールの匿名化データを蓄積することで、自動生成精度が向上する。  
後発が追いつけないデータ資産になる。

例:
- 「EC サイトで頻出するテスト条件 TOP20」
- 「認証フローの典型的な入力項目パターン」

### モート 2: 業種別テンプレートライブラリ

業種 × 機能パターンを製品に組み込む:

| 業種 | テスト観点セット |
|---|---|
| EC サイト | カート・決済・在庫・セッション |
| 金融・銀行 | 振込・ログイン・セッション・暗号化 |
| 医療 | 入力制限・アクセス制御・監査ログ |
| 官公庁 | JIS X 8341 アクセシビリティ |

「業種名を選ぶだけで最適なテスト観点が出る」体験が、業界標準ツールとして定着させる。

### モート 3: AutoRun × CI/CD の「常時監視」サービス

```
週 1 回の定期クロール
→ ドリフト検知
→ Slack アラート
→ 自動テスト実行
= 「QA サービスのサブスクリプション」として位置づけ
```

Datadog がインフラ監視を独占したように、  
WebSpec2Doc が「Web 仕様の常時監視」というカテゴリを定義する。

## 9. リスクと対策

| リスク | 影響度 | 対策 |
|---|---|---|
| Playwright 本家が Spec 生成機能を追加 | 高 | 業種テンプレート・LLM 統合で差別化を先行 |
| 大手 SaaS（Mabl 等）が日本語対応強化 | 中 | 検証会社との深い関係と日本語 UX で守る |
| クロール対象サービスが拒否 | 低 | User-Agent 管理、robots.txt 遵守、レート制限 |
| セキュリティ懸念（認証情報管理） | 高 | SOC2 取得・暗号化徹底・ゼロ知識設計 |
| 開発リソース不足 | 高 | Phase 1（課金基盤）に集中、残りは後回し |

## 10. 結論：最速ロードマップ

```
Month 0-3:  SaaS 化（課金・認証・クラウド）← 最大のボトルネック
Month 3-6:  日本 QA コミュニティで PLG 浸透
Month 6-12: 検証会社 5 社とエンタープライズ契約
Month 12-18: 英語化・グローバル PLG 開始
Month 18-24: $1M ARR 達成

キャッシュフロー目安:
  Month  6: MRR ¥500K   ($3.5K)
  Month 12: MRR ¥3M     ($20K)   ← エンタープライズ 1〜2 社入れば加速
  Month 18: MRR ¥8M     ($53K)
  Month 24: MRR ¥15M    ($100K)  = ARR $1.2M ✓
```

**最も重要な 1 つのアクション**:  
`python app.py` で起動するローカルツールを、今日から「URL を渡すだけで動く Web サービス」に変える。SaaS 化が全ての起点。

技術的には現在 85% 完成しており、残り 15%（SaaS 基盤・課金・チーム機能）が商用化の唯一の壁。  
プロダクト自体の差別化軸は既に確立されているため、GTM と SaaS 化のスピードが勝負を決める。
