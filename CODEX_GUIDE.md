# Claude × Codex 実践ガイド

このプロジェクトで得た知見 + 公式ドキュメント調査をまとめたもの。

---

## 役割分担（最重要）

| 担当 | 仕事 |
|------|------|
| **Claude** | 計画・設計・タスク分割・コードレビュー・最終確認・意思決定 |
| **Codex** | 大規模実装・複数ファイル一括編集・長時間実行タスク |

> Claude が設計図を描き、Codex が現場で建てる。確認は必ず Claude が行う。

---

## Codex に投げるプロンプトの書き方

### テンプレート
```
## 作業ディレクトリ
/絶対パス/で/指定すること

## 実装対象
ファイルごとに何をするか明示。新規作成か編集かを区別。

## コーディングルール（厳守）
- 型ヒント必須
- 1関数50行以内
- etc.

## 完了条件（検証可能な形で書く）
1. ファイル X が存在する
2. `python -c "import X; print('ok')"` が通る
3. pytest 全 PASS
```

### 書き方のコツ
- **絶対パス**で作業ディレクトリを指定（相対パスは迷子になる）
- **完了条件はコマンドで検証できる形**にする
- 削除していいファイル・関数は「削除して構いません」と明示
- 既存コードを渡すなら「現在の実装」セクションを設けてコピペ

## ファイルが書き込まれない問題

### 原因
Codex がサンドボックス環境で実行し、ホストのファイルシステムに反映されない場合がある。

### 確認方法
```bash
# タスク完了後、実際にファイルが変わったか確認する
ls src/generator/
git diff --stat  # git管理下なら差分を見る
```

### 対策
1. `/codex:rescue --fresh` で新しいスレッドとして投げ直す
2. Codex の出力サマリーに「実際に書き込みました」とあっても過信しない
3. Codex が失敗したら **Claude が直接実装する**（今回の html_reporter.py がその例）

---

## 並列処理パターン

### Claude + Codex 同時実行
```
Claude: E2Eテスト実行（自分でやる）
Codex:  HTMLレポーター実装（バックグラウンドで投げる）
```
→ 互いに別ファイルを触る独立タスクなら問題なし。

### 注意
- 同じファイルを両方が触ると競合する
- Codex タスクは `--background` なしでも非同期で走る（完了通知で戻ってくる）

---

## リトライ戦略

```
1回目失敗 → /codex:rescue --resume でコンテキストを引き継いで修正指示
2回目失敗 → /codex:rescue --fresh で白紙から投げ直す
3回目失敗 → Claude が直接実装する（時間的に同等かそれ以下）
```
