WebSpec2Doc ユーザーガイド

URL解析からQAプロセス成果物の作成まで、画面上で確認できる操作ガイドです。

1. サイトを追加左上の「サイトを追加」から対象URLを入力します。
2. 画面分析「画面分析」で取得対象ページを検出します。
3. 条件設定対象画面、ログイン、差分出力を確認します。
4. レポート確認概要、画面別仕様、入力項目、遷移、履歴差分を確認します。
5. QAプロセス解析済みサイトを選び、Step by StepでQA成果物を作成します。
6. モデル/カバレッジ画面遷移グラフとTrace IDの網羅性を確認します。
7. 自動テスト候補Playwright化しやすい候補をレビューします。
8. 品質観点CSV、a11y、OWASP、境界値、API契約の観点を確認します。

レポートの読み方 — どのタブで何が分かるか

タブ何が分かるか使い方のポイント
テスト条件 画面・項目ごとに機械導出したテスト条件（必須チェック・境界値・選択肢・形式）を一覧表示 「必須のみ」フィルターで最小テストケース候補を絞り込む。CSVで書き出してテストケース設計のたたき台に使う
設計 各画面へのテスト技法推奨マトリクス（同値分割・境界値・状態遷移・決定表など8技法） バッジをホバーすると推奨理由が出る。技法名をクリックすると技法ガイドパネルが開く
技法詳細 画面別に推奨した技法の根拠（遷移先数・フィールド種別・制約）を文章で説明 「なぜこの技法が推奨されたか」を説明する資料として設計レビューに使える
画面遷移図 シーケンス図・コミュニケーション図・アクティビティ図・テスト観点マップの4形式 ↓SVG / ↓PNG / ↓MMD ボタンで図をダウンロードし、設計書・報告書に貼り付けられる
履歴・差分 再クロール前後の画面追加・削除・フィールド変更を検知 リリース後にリクロールすると仕様ドリフトが可視化される。バッジの数字が差分件数

テスト技法が推奨されたら — 実際の作業フロー

推奨技法典型的な状況次のアクション
同値分割入力項目に型・形式制約あり（例: メール形式、数値のみ）有効クラス1件＋無効クラス1件以上をテストケース化
境界値分析min/max 制約あり（例: 1〜9泊、0〜99人）最小値・最大値・境界±1 の計4〜6ケースを作成
状態遷移ページに複数の遷移先（予約→確認→完了など）有効遷移＋戻り遷移＋エラー経路をテスト
決定表複数条件の組み合わせ（例: ログイン有無×会員種別）条件の真偽組み合わせを表で整理してケース化
ペアワイズ独立した選択肢が3種以上（例: 朝食×禁煙×部屋タイプ）全組み合わせではなくペアを網羅する最小セットで実施

4ステップURL解析

Step画面操作
1URL入力・解析クロール対象URLを入力し、「画面分析」を押します。
2条件設定取得対象画面、ログイン、差分出力を確認します。
3実行受付、ページ取得、解析、結果整理の進捗を確認します。
4レポート生成された仕様、入力項目、遷移、差分、出力形式を確認します。

QAプロセス

Step画面内容
1対象サイト選択解析済みサイトを選びます。対象の report.json が必要です。
2入力仕様サマリー画面、フォーム、入力項目、必須項目、操作要素を確認します。
3テスト計画スコープ、テストレベル、質問待ち事項を整理します。
4テスト分析画面別にリスク源と確認対象を洗い出します。
5テスト設計観点IDと元仕様の対応を確認します。
6テストケースTrace ID付きのケースを確認します。
7横断レビュー / レポート欠落リスクを確認し、QAプロセスレポートを生成します。

OpenAI補完とCSV観点

  • 標準: OpenAI APIを呼ばず、追加料金なしでローカル生成します。
  • 補完ON: 設定済みのOpenAI APIキーでStep 3〜7を構造化JSONとして補完します。
  • CSV観点: SaaS/Webテスト観点CSVをテスト分析・設計・ケース・横断レビューに反映します。

CI/CD 連携（仕様ドリフトの自動監視）

デプロイのたびに仕様ドリフトを自動検知できます。差分があると exit code 1 で終了するため、パイプラインで検知可能です。

  • CLI: python src/main.py --url https://staging.example.com --compare --fail-on-drift
  • GitHub Actions: リポジトリ同梱の .github/workflows/spec-drift.yml をそのまま利用できます（毎週月曜の定期実行 + 手動実行）。リポジトリ変数 WEBSPEC_TARGET_URL に対象URLを設定してください。
  • Slack通知: 設定 → 通知タブで Incoming Webhook URL を登録すると、ドリフト検知時に自動でSlackへ通知します。
  • 社内環境のクロール: ステージング等のプライベートアドレスを対象にする場合は環境変数 WEBSPEC2DOC_ALLOW_LOCAL=1 を設定してください（既定では SSRF 保護のため拒否されます）。

よくある確認

  • QAプロセスに対象サイトが出ない場合は、先にサイトを解析して report.json を生成してください。
  • ログインが必要な画面は、条件設定で認証情報を入力してから解析してください。
  • 前回との差分は「再クロール」後、レポートの「履歴・差分」タブで確認できます。
