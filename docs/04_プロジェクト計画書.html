リスク管理
IDリスク影響度発生確率対策
R-01 SPA / JavaScript ヘビーなサイトでクロールが止まる 高 中 networkidle 待機 + クリックイベント発火を試みる。不可能な場合は警告ログを出しスキップ
R-02 対象サイトの規約違反（過剰クロール） 高 低 robots.txt を遵守する実装を追加。ページ間に 1〜2 秒のウェイトを挿入。社内判断で使用先を制限する。
R-03 LLM コストの超過 中 中 Claude Haiku を使用（最安モデル）。キャッシュで重複処理を排除。1 プロジェクトあたりの上限コストを設定する。
R-04 大規模サイトで処理時間が5分を超える 中 中 max_pages・depth のデフォルト制限で防止。超過時は途中経過を保存して再開できる仕組みを将来実装。
R-05 ベリサーブ CSS のブランドカラーが正式値と異なる 低 高 CSS 変数（:root）で一元管理しているため、veriserve.css の変数値を修正するだけで全文書に反映される。
