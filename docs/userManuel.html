1このツールについて

解決する課題

アジャイル開発・DevOps 現場での第三者検証支援において、次のような状況が頻繁に起こります。

「ドキュメントがないので、まず触って覚えてください」

これではテスト設計の網羅性が担保できません。WebSpec2Doc はウェブサイトを自動でクロールし、QA テスト設計のインプットとなる文書を即時生成することで、初日からテスト設計を開始できる状態を作ります。

生成されるもの

成果物内容QA での用途
画面一覧検出した全画面の URL・タイトル・フォーム数テスト対象範囲の確定
画面遷移図画面間の移動経路を図示遷移テスト・シナリオ設計
入力項目仕様フォームのフィールド・型・必須可否境界値テスト・バリデーション確認
スクリーンショット各画面の自動撮影画像テスト前確認・エビデンス比較
HTML レポート上記をまとめた単一 HTML ファイルチームへの共有・顧客提出

対象サイト

対応：公開されているウェブサイト・SPA（JavaScript レンダリング対応）
非対応：ログインが必要なページ（認証前ページまでは到達可能）

2動作環境

項目要件
OSmacOS / Linux / Windows
Python3.11 以上
ネットワーククロール対象サイトへの HTTP/HTTPS アクセス
ディスクスクリーンショット保存用に数十 MB（サイト規模による）

3インストール

  1. 仮想環境を作成・有効化 
    python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
  2. 依存パッケージをインストール 
    pip install -r requirements.txt
  3. Playwright ブラウザをインストール 
    playwright install chromium
  4. 動作確認 
    python src/main.py --help

    「usage: main.py [-h] --url URL ...」と表示されれば成功です。

4基本的な使い方

⚡ 最小構成

python src/main.py --url https://example.com

⭐ 推奨（HTML レポート付き）

python src/main.py \
  --url https://example.com \
  --depth 2 \
  --max-pages 30 \
  --format md,html

📊 全形式出力（Excel 含む）

python src/main.py \
  --url https://example.com \
  --depth 2 \
  --max-pages 30 \
  --format md,html,excel

実行後の流れ

コマンド実行
→
ブラウザ起動・クロール
→
解析・グラフ生成
→
output/ に文書生成

完了すると output/{ドメイン名}/ に成果物が生成されます。

5オプション詳細

オプションデフォルト説明
--url 必須 — クロールを開始する URL。https:// を含めてください。
--depth 3 リンクを辿る深さ。初回は 2 を推奨。
--max-pages 50 クロールする最大ページ数。大規模サイトでは 20〜30 に制限推奨。
--output ./output 出力先ディレクトリ。実際には {output}/{ドメイン}/ に生成。
--format md md / html / excel をカンマ区切りで指定。

--depth の目安

depthクロール範囲目安ページ数推奨場面
1起点 URL のみ1〜数ページ単一ページの確認
2直接リンク先まで10〜30 ページ初回調査・中規模サイト
3さらにその先まで30〜100 ページ網羅的調査

6出力ファイルの見方

screens.md — 画面一覧

Markdown テーブル形式で全画面を一覧化します。

列名内容
画面IDP001 から連番。テスト仕様書の「対象画面」として使用できます。
URLそのページのパス部分
タイトル<title> タグから取得
フォーム数その画面で検出されたフォームの数
遷移先リンクされている他の画面 ID

forms.md — フォーム・入力項目一覧

列名内容
画面IDフォームがある画面の ID
フィールド名name 属性の値
型text / email / password / select / textarea など
必須required 属性の有無（Yes / No）
placeholder入力ヒントテキスト

transition.mmd — 画面遷移図（Mermaid 形式）

ナビゲーション共通リンクを自動除外し、意味のある遷移のみを表示します。Mermaid Live Editor に貼り付けると図として描画できます。

report.html — 統合 HTML レポート

上記すべてを 1 ファイルにまとめた HTML レポートです。ブラウザで開くと各画面のスクリーンショット・遷移図・テーブルを確認できます。ファイル 1 つをメール・チャットで共有できます。

ヒント：Mermaid 遷移図のレンダリングにはインターネット接続が必要です（CDN を使用）。

spec.xlsx — Excel 仕様書

シート名内容
Screens画面一覧（画面ID / URL / タイトル / フォーム数）
Forms入力項目一覧（画面ID / URL / フィールド名 / 型 / 必須 / placeholder）

7QA テスト設計への活用方法

テスト設計の流れ

WebSpec2Doc 実行（5〜10分）
↓
screens.md で画面一覧を把握
→ テスト対象範囲を確定
↓
transition.mmd で遷移経路を確認
→ シナリオテストを設計
↓
forms.md で入力項目を整理
→ 境界値・バリデーションテスト設計
↓
テスト仕様書・テストケース作成

フォーム情報とテスト設計の対応

取得できる情報設計すべきテストテスト技法
型: email@ なし・不正形式での送信同値分割・境界値
型: password最小長・最大長・使用禁止文字境界値分析
型: text最大長超過・特殊文字・SQL インジェクション境界値・セキュリティ
必須: Yes空欄のまま送信同値分割（無効クラス）
必須: No任意項目が空でも送信できるか同値分割（有効クラス）
placeholderヒントテキスト消去・アクセシビリティUI テスト

遷移図の活用

  • 遷移テスト：図に表れているすべての矢印がテストすべき遷移経路
  • デッドリンク確認：遷移先が存在しないノードの検出
  • 異常系：直接 URL アクセス・ブラウザの戻るボタンでの挙動

8トラブルシューティング

症状原因対処方法
SSL 証明書エラーの警告 macOS の Python が SSL 証明書を参照できない /Applications/Python 3.x/Install Certificates.command を実行。クロール自体は継続するため無視も可。
ページ数が少ない・クロールされない depth が浅い / ログインが必要なページ --depth を増やす。認証ページは対象外。
タイムアウトが頻発する サーバーが重い / ネットワーク遅延 --max-pages を減らしてページ数を絞る。
遷移図（Mermaid）が表示されない オフライン環境 / CDN 未アクセス transition.mmd を mermaid.live に貼り付ける。
report.html のサイズが大きい スクリーンショットが多い / ページ数が多い 500KB 超のスクショは自動スキップ。--max-pages を減らすと解消。
ModuleNotFoundError 仮想環境が有効でない / pip install 未実施 source venv/bin/activate → pip install -r requirements.txt
