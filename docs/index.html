プロジェクト概要
背景・課題

アジャイル開発・DevOps 現場への第三者検証支援において、「仕様書・設計書がないので触って覚えてください」という状況が頻繁に発生している。ドキュメントのない状態ではテスト設計の網羅性が担保できず、品質保証専門家としての価値が発揮できない。

提案ソリューション

公開 Web サービスの URL を入力するだけで、QA エンジニアがテスト設計を始めるために必要な文書（画面一覧・画面遷移図・入力項目仕様）を自動生成するツールを開発する。

目標： 「URL を渡したら 5 分以内に、テスト設計を始められる最低限の文書一式が生成される」
主要指標（KPI）
テスト設計開始までの時間
1日目から
文書生成時間
5分以内 / 50画面
対象ページ数
50画面 / 実行
想定工数削減
2〜4人日 / プロジェクト

クイックスタート
環境構築
cd 008_CrateSpec2HTML
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
playwright install chromium
cp .env.example .env
# .env を編集して ANTHROPIC_API_KEY を設定
実行例
# シンプル実行（LLM なし・無料）
python src/main.py --url https://example.com

# AI 解析付き（テスト観点を自動推定）
python src/main.py --url https://example.com --llm

# 全形式で出力
python src/main.py --url https://example.com --llm --format html,md,excel
