# プラットフォーム構成と AutoRun 入口の設計

- 状態: **実装済み**（2026-07-20）
- 対象: ログイン後の導線全体、および AutoRun の受付・実行画面
- 制約: **PC 専用**（レスポンシブ/モバイル対応は行わない）

> 本書は検討過程ではなく、**実装した設計**を記録するものです。検討途中で採用しなかった案
> （テスト目的プリセット、ドキュメント作成と AutoRun の入口統合）は破棄されました。

---

## 1. 全体構成 — ログイン → システム選択 → 各システム

```text
ログイン (/auth/login)
   ↓ 認証成功（next 既定 = /systems）
システム選択 (/systems)
   ├─ System 01: ドキュメント作成 → /generate
   └─ System 02: AutoRun          → /auto-run
```

**AutoRun はドキュメント作成とは独立したシステム**として並立させる。両者を統合したり入れ子にしたりしない。

- 共通するのは**ログインとアカウントのみ**。入口・世界観・ナビゲーションは独立。
- システム選択画面は **2 カード並列**。ドキュメント作成＝青系、AutoRun＝緑系で世界を色分けする。
- 実装: `templates/system-select.html` / `static/css/system-select.css` / `web/routes/pages.py` の `/systems`。
- 認証は `auth_guard`（`before_request`）が全ルートに適用されるため、`/systems` も自動的に保護される。

### システムスコープ（相手システムを混ぜない）

`static/js/system-scope.js` が現在の系を判定し、サイドバーを絞り込む。

- 判定順: **現在のパス** → `sessionStorage` → 既定 `docs`
- ナビ項目は `data-system` 属性で所属を宣言する

| 所属 | 画面 |
|---|---|
| `docs` | ホーム / 新規解析 / テストケース / 品質観点 / 観点管理 |
| `autorun` | AutoRun / 実行履歴 |
| `common` | ユーザーガイド / 参考 / 設定 |

- 対象外の項目と、中身が空になったグループ見出しは非表示にする。
- サイドバー上部に**システム切替 UI**（現在の系名＋「切替」→ `/systems`）を置く。ブランドロゴのリンク先も系のホームへ追従する。

## 2. AutoRun の入口 — 単一フロー

**2 ペイン（左フォーム＋右メイン）を廃止**し、中央寄せの単一カラム（`.autorun-single`）に変更した。
1 枚の画面が状態に応じて姿を変える。

```text
[受付]  #autorun-idle-msg
          ├ 見出し（URL か仕様書を渡すだけ）
          ├ #autorun-form-area （モード / URL / 文書 / 観点 / 詳細 / 開始）
          ├ これから走る 11 工程（見立て・設計・実行納品）
          ├ 納品物チップ
          └ 報告の範囲（claim_scope）
   ↓ 開始
[実行]  進捗ステッパー → ライブプレビュー → 実行中テスト → 実行ログ
   ↓ 完了
[納品]  完了カード / 成果物 / 最近の実行
```

**設計上の要点**: `#autorun-form-area` を `#autorun-idle-msg` の**内側**に置いた。
これにより既存 JS の `_autorunShowRunning()` が `#autorun-idle-msg` を隠すだけで、
フォームごと受付全体が実行ビューへ切り替わる。**JS の変更は不要**。

## 3. AutoRun の思想（画面に反映しているもの）

- **委任**: URL か仕様書を渡すだけ。技法・観点・深さはシステム側が判断し、ユーザーに選ばせない。
- **フルQA 既定**: 範囲を絞らない。人手では到達できない網羅をシステムだから当てる。
- **止めない**: 観測で決まらない事項（対応ブラウザ・認証ロール・副作用・合否基準）は着手前ゲートにせず、
  前提を置いて走り切り、レポートに「仮定・要確認」として明示する。
- **事実と判断の分離**: 観測事実と品質判断を層で分け、最終の受入判断は人間に残す。
- **不在は証明しない**: 出せるのは「自動検出できた範囲で観測した事実」のみ。
  検証できなかった領域は「非該当」ではなく **「未検証」** と明記する。
  現新比較は前回スナップショットがある場合のみ成立し、初回は基準確立のみ。

この方針の裏付けは `docs/research/prior-research-survey.md`（Barr らのオラクル問題、
Vigo らによる自動アクセシビリティ検査のカバレッジ実測 最大 32.4%）を参照。
