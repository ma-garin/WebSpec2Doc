## ポジショニング

「AI テストエージェントへのスペック供給者（Spec Supplier）」

テスト自動化ツール（Playwright / Selenium / AI エージェント）は、
「画面がどこにあるか・何が入力できるか」を自分では知れない。
WebSpec2Doc はその「地図」を提供するミドルレイヤーとして位置づける。

競合との差別化：
- Screaming Frog, Sitebulb → SEO 目的。QA 観点なし
- Playwright codegen → 操作録画。画面体系の俯瞰なし
- 手動 Excel 仕様書 → 時間コスト大、陳腐化が早い

WebSpec2Doc の空白地帯：
「ゼロ文書の現場で、初日からテスト設計を始められる QA 特化ドキュメント自動生成」


## マネタイズの楔（最優先実装）

### スペックドリフト検出（Spec Drift Detection）

仕組み：
  1. クロール結果を snapshot として保存（SQLite or JSON）
  2. 定期または手動で再クロール
  3. 前回との差分を検出：新規画面 / 削除画面 / フォーム変更 / リンク変更
  4. 差分レポートを生成（HTML + Slack/Teams 通知）

なぜこれが楔か：
- 継続利用を生む（一度使ったら定期的に使い続ける）
- 「変更検知 → テスト範囲特定」という明確な ROI
- CI/CD パイプラインへの組み込みが自然
- 競合に存在しない QA 特化機能

実装優先度：★★★★★


## 競合優位（モート）

### オンプレミス / ローカル実行

企業が SaaS にデータを送れない理由：
- 社内システム（ログイン必要、VPN内）
- 金融・医療・官公庁の情報セキュリティポリシー
- 第三者検証会社が扱う顧客企業の画面情報

戦略：
- CLI ツールとして完全ローカル動作を維持（現行アーキテクチャが正解）
- Docker イメージで環境差異を吸収
- エンタープライズ向けに「データ一切外部送信なし」を明示


## プロダクト深化ロードマップ

### 優先度 HIGH

1. ログイン対応
   - Playwright で認証フローを記録・再生
   - 環境変数で認証情報管理（.env）
   - 実現できれば「対象外サイト」がゼロになる

2. スペックドリフト検出（上述）

3. LLM テスト観点生成
   - 各画面に「QA テスト観点 3 点」を自動付与
   - Claude Haiku で低コスト処理（100 ページ ≒ $0.01）
   - 入力項目から「推定バリデーションルール」を推論
   - src/llm/ は __init__ のみ（未実装）。env 準備 + 実装が必要

### 優先度 MEDIUM

4. アクセシビリティ検査
   - Playwright の axe-core 統合
   - WCAG 2.1 AA 違反を画面ごとに一覧化
   - 日本語では競合ほぼなし

5. TestRail / Jira 直接連携
   - API 経由で画面一覧 → テストケース自動作成
   - Excel 出力の上位互換

6. AGENTS.md 生成
   - AI テストエージェント向けにサイト構造を machine-readable で出力
   - 新興フォーマット（競合皆無）

### 優先度 LOW（将来）

7. VS Code 拡張
   - 開発中に即クロール → 仕様確認
   - QA と開発が同一ツールを使う世界

8. GitHub Actions 統合
   - PR マージ後に自動クロール → ドリフト検出
   - 「壊れた画面」を CI で検出


## マネタイズモデル案

| プラン | 対象 | 価格帯 | 提供形態 |
|--------|------|--------|---------|
| Free CLI | 個人 QA エンジニア | 無料 | OSS（GitHub） |
| Pro | フリーランス・小規模チーム | ¥3,000〜5,000/月 | SaaS or ライセンス |
| Enterprise | 第三者検証会社・大企業 | 要見積 | オンプレ + サポート |

収益の核：Enterprise オンプレライセンス
理由：第三者検証会社はクライアント情報を外部送信できない →
      SaaS 版は使えず、ローカル CLI が唯一の選択肢になる


## 次の 3 手（実装順序）

1. **スペックドリフト検出**（v1.1 目標）
   - snapshot 保存 → 差分比較 → 差分レポート HTML
   - これで「使い捨てツール」から「継続利用ツール」に変わる

2. **ログイン対応**（v1.2 目標）
   - Playwright セッション管理
   - 「ログイン必要サイト対象外」制約を撤廃

3. **LLM テスト観点生成**（v1.3 目標）
   - --llm フラグ ON で各画面に AI 推奨テスト観点を付与
   - Claude Haiku / キャッシュ使用でコスト最小化
